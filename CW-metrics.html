CloudWatch ▼
Logs Metrics Metric Statistics

a) List Metrics

A metric tracks performance in your AWS account—like EC2 CPU usage, Lambda invocations, network traffic, or website errors—acting as a health check for your resources.

Command:

aws cloudwatch list-metrics

This command is used to retrieve a list of available CloudWatch metrics for your AWS account. This includes metrics automatically collected from AWS services like EC2, RDS, Lambda, etc., and also any custom metrics you've pushed using put-metric-data.

Optional parameters you can add:

Parameters Description
--namespace (where the metric lives) Filter by AWS service namespace (e.g., AWS/EC2)
--metric-name (what you're measuring) Filter by a specific metric name (e.g., CPUUtilization)
--dimensions ( Who or what the measurement is for) Filter by specific dimensions (e.g., Name=InstanceId,Value=i-1234567890abcdef0)
--recently-active Shows only metrics that have had data within the last 3 hours (PT3H)

b) List only EC2 metrics

Command:

aws cloudwatch list-metrics --namespace AWS/EC2

c)List only CPU-Utilization metric across all services

Command:

aws cloudwatch list-metrics --metric-name CPUUtilization

d) List metrics for a specific EC2 instance

Command:

aws cloudwatch list-metrics \
   --namespace AWS/EC2 \
   --metric-name CPUUtilization \
   --dimensions Name=InstanceId,Value=i-1234567890abcdef0

e) List recently active metrics

Command:

aws cloudwatch list-metrics --recently-active PT3H

“it Show only the metrics that have had data points within the last 3 hours.”
